
# E-Commerce Sales Data Analysis SQL


## Project Overview


The E-Commerce Sales Data Analysis project is designed to demonstrate the use of SQL for extracting, managing, and analyzing business data. The project uses an e-commerce database containing customer, product, order, and sales information. Various SQL queries are performed to generate meaningful insights that help understand customer behavior, product performance, and sales trends.

## Business Objectives

The primary objective of this project is to use SQL queries for data analysis and reporting. The project covers essential SQL concepts such as data retrieval, filtering, sorting, grouping, joins, subqueries, aggregate functions, views, and query optimization.

## Tasks Performed

Data Retrieval Using SELECT

Retrieved records from database tables to view available information.

2. Data Filtering Using WHERE

Filtered records based on specific conditions such as product price and customer location.

3. Data Sorting Using ORDER BY

Sorted product and sales data in ascending and descending order.

4. Data Grouping Using GROUP BY

Grouped products by category and calculated category-wise statistics.

5. JOIN Operations

Performed:

INNER JOIN
LEFT JOIN
RIGHT JOIN

These joins were used to combine data from multiple tables and generate meaningful reports.

6. Subqueries

Implemented nested queries to retrieve advanced business insights, such as customers who placed orders.

7. Aggregate Functions

Used SQL aggregate functions:

SUM()
AVG()
COUNT()
MAX()
MIN()

These functions helped analyze sales performance and product statistics.

8. Views Creation

Created SQL views to simplify complex queries and support business reporting.

9. Query Optimization

Created indexes on frequently used columns to improve query execution speed and database performance.
